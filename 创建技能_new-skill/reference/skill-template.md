## 字段说明

### Front Matter (YAML)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | Skill 标识符，只能包含字母、数字、连字符 |
| `command` | string | ✅ | 触发命令（不含 /） |
| `user_invocable` | boolean | ✅ | 是否可被用户通过命令触发 |
| `description` | string | ✅ | 描述何时使用（< 1024 字符） |

### description 写法指南

**核心原则**：描述"何时触发"，而非"做什么"

```yaml
# ❌ 错误 - 描述做什么
description: 实现测试驱动开发，确保代码质量

# ✅ 正确 - 描述何时用
description: 在编写任何功能或 bug 修复前使用，确保先写失败测试再实现
```

**原因**：description 用于自动触发判断，必须包含触发条件。

## 命名规范

### Skill 名称 (name)

- 只用小写字母、数字、连字符
- 使用动词或动词短语
- 描述性，见名知义

**示例**：
- ✅ `test-driven-development`
- ✅ `systematic-debugging`
- ❌ `TDD` （太短）
- ❌ `my_skill` （不能用下划线）

### 命令 (command)

- 简短易记
- 常用缩写可接受

**示例**：
- `tdd` → test-driven-development
- `debug` → systematic-debugging
- `review` → two-phase-review
