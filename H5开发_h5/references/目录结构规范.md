## 二、目录命名规范

### 2.1 页面目录

| 规范 | 示例 |
|------|------|
| 模块名小写 | `tenant/`、`contract/` |
| 功能名小写 | `list/`、`detail/`、`form/` |
| 页面文件 PascalCase | `TenantList.vue`、`ContractDetail.vue` |

### 2.2 组件目录

| 规范 | 示例 |
|------|------|
| `comp-` 前缀 | `comp-popup/`、`comp-empty/` |
| 组件文件同名 | `comp-popup/comp-popup.vue` |
| 子组件放目录内 | `comp-popup/components/xxx.vue` |

### 2.3 Store 目录

| 规范 | 示例 |
|------|------|
| 模块名 PascalCase | `Common.js`、`TenantManagement.js` |
| 导出函数 use 前缀 | `useCommonStore`、`useTenantStore` |

## 三、文件组织原则

### 3.1 按功能模块组织页面

```
pages/
├── tenant/                 # 租客模块
│   ├── list/              # 列表
│   ├── detail/            # 详情
│   ├── form/              # 表单
│   └── components/        # 模块私有组件
│       └── TenantCard.vue
```

### 3.2 按类型组织工具函数

```
utils/
├── request.js             # HTTP 请求
├── storage.js             # 本地存储
├── validate.js            # 表单验证
├── format.js              # 数据格式化
├── platform.js            # 平台判断
└── constants.js           # 常量定义
```

### 3.3 按业务组织 API

```
api/
├── index.js               # 统一导出
└── modules/
    ├── tenant.js          # 租客 API
    ├── contract.js        # 合同 API
    └── common.js          # 公共 API
```

## 四、组件存放规则

| 组件类型 | 存放位置 | 命名 |
|---------|---------|------|
| **全局公共组件** | `/components/comp-xxx/` | `comp-popup`、`comp-empty` |
| **模块私有组件** | `/pages/xxx/components/` | `XxxCard.vue`、`XxxItem.vue` |
| **页面子组件** | 与页面同目录 | `components/StepBasic.vue` |

## 六、禁止的目录结构

```
❌ 错误示例

components/
├── Popup.vue              # 缺少 comp- 前缀
├── popup/                 # 缺少 comp- 前缀
└── comp-popup.vue         # 未放在独立目录

pages/
├── tenantList.vue         # 应放在 tenant/list/ 目录
└── tenant-detail.vue      # 文件名应使用 PascalCase

utils/
└── index.js               # 工具函数不应合并到一个文件
```

## 七、检查清单

新建模块前确认：

- [ ] 页面放在 `pages/{模块名}/{功能}/` 目录
- [ ] 公共组件使用 `comp-` 前缀
- [ ] 模块私有组件放在模块目录下的 `components/`
- [ ] Store 文件放在 `store/modules/`
- [ ] API 文件放在 `api/modules/`
- [ ] 静态资源按类型分目录存放
